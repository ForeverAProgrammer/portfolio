Gradle Build Tool Guide

Gradle is a powerful and flexible build automation tool used primarily for Java projects. This guide covers essential concepts including multi-project builds.

What is Gradle?â€‹

Gradle is a build automation tool that:

  • Compiles source code
  • Manages dependencies
  • Runs tests
  • Packages applications
  • Publishes artifacts

Key Features:

  • Flexible and powerful
  • Fast incremental builds
  • ğŸ—ï¸ Multi-project support
  • Extensive plugin ecosystem
  • Kotlin or Groovy DSL

Installationâ€‹

Using SDKMAN (Recommended)â€‹

sdk install gradle
gradle --version

Manual Installationâ€‹

  1. Download from gradle.org
  2. Extract archive
  3. Add GRADLE_HOME/bin to PATH

Check Installationâ€‹

gradle --version

Basic Gradle Conceptsâ€‹

Build Scriptâ€‹

Main configuration file: build.gradle (Groovy) or build.gradle.kts (Kotlin)

Tasksâ€‹

Units of work that Gradle executes:

gradle build      # Build project
gradle test       # Run tests
gradle clean      # Clean build directory

Pluginsâ€‹

Extend Gradle functionality:

plugins {
    id 'java'
    id 'application'
}

Dependenciesâ€‹

External libraries your project needs:

dependencies {
    implementation 'com.google.guava:guava:31.1-jre'
    testImplementation 'junit:junit:4.13.2'
}

Multi-Project Buildsâ€‹

Multi-project builds allow you to organize large applications into smaller, modular projects.

Project Structureâ€‹

my-application/
â”œâ”€â”€ settings.gradle
â”œâ”€â”€ build.gradle
â”œâ”€â”€ app/

â”‚   â””â”€â”€ src/
â”œâ”€â”€ utils/
â”‚   â”œâ”€â”€ build.gradle
â”‚   â””â”€â”€ src/
â””â”€â”€ api/

Root settings.gradleâ€‹

rootProject.name = 'my-application'

// Include subprojects
include 'app'
include 'core'
include 'utils'
include 'api'

Root build.gradleâ€‹

// Common configuration for all projects
subprojects {
    apply plugin: 'java'

    group = 'com.example'
    version = '1.0.0'

    java {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }

    repositories {
        mavenCentral()
    }

    dependencies {
        // Common dependencies for all subprojects
        testImplementation 'junit:junit:4.13.2'
        testImplementation 'org.mockito:mockito-core:5.3.1'
    }
}

// Configuration for all projects (including root)
allprojects {
    repositories {
        mavenCentral()
    }
}

Subproject build.gradle (app/build.gradle)â€‹

plugins {
    id 'application'
}

dependencies {
    // Depend on other subprojects
    implementation project(':core')
    implementation project(':utils')
    implementation project(':api')

    // External dependencies
    implementation 'org.springframework.boot:spring-boot-starter-web:3.1.0'
}

application {
    mainClass = 'com.example.app.Application'
}

Subproject build.gradle (core/build.gradle)â€‹

dependencies {
    implementation project(':utils')
    implementation 'com.google.guava:guava:31.1-jre'
}

Subproject build.gradle (utils/build.gradle)â€‹

dependencies {
    implementation 'org.apache.commons:commons-lang3:3.12.0'
}

Subproject build.gradle (api/build.gradle)â€‹

dependencies {
    implementation project(':core')
    implementation 'com.fasterxml.jackson.core:jackson-databind:2.15.0'
}

Multi-Project Commandsâ€‹

# Build all projects
gradle build

# Build specific subproject
gradle :app:build
gradle :core:test

# Run app subproject
gradle :app:run

# Build multiple specific projects
gradle :core:build :utils:build

# View all projects

# View dependencies of specific project
gradle :app:dependencies

# Run tests in all projects
gradle test

# Run tests in specific project
gradle :core:test

# Clean all projects
gradle clean

# Clean specific project
gradle :app:clean

Dependency Configurationsâ€‹

dependencies {
    // Compile-time and runtime (visible to consumers)
    api 'com.google.guava:guava:31.1-jre'

    // Compile-time and runtime (not visible to consumers)
    implementation 'org.apache.commons:commons-lang3:3.12.0'

    // Runtime only
    runtimeOnly 'mysql:mysql-connector-java:8.0.33'

    // Compile-time only
    compileOnly 'org.projectlombok:lombok:1.18.28'
    annotationProcessor 'org.projectlombok:lombok:1.18.28'

    // Test dependencies
    testImplementation 'junit:junit:4.13.2'
    testRuntimeOnly 'org.junit.platform:junit-platform-launcher'
}

Gradle Wrapperâ€‹

The wrapper ensures everyone uses the same Gradle version.

Generate Wrapperâ€‹

gradle wrapper --gradle-version 8.5

Files Createdâ€‹

gradle/
â””â”€â”€ wrapper/
    â”œâ”€â”€ gradle-wrapper.jar
    â””â”€â”€ gradle-wrapper.properties
gradlew          # Unix/Mac script
gradlew.bat      # Windows script

Using Wrapperâ€‹

./gradlew build      # Unix/Mac
gradlew.bat build    # Windows

Always commit wrapper files to version control!

Custom Tasksâ€‹

Simple Taskâ€‹

tasks.register('hello') {
    doLast {
        println 'Hello, Gradle!'
    }
}

Run: gradle hello

Task with Dependenciesâ€‹

tasks.register('deploy') {
    dependsOn 'build', 'test'

    doLast {
        println 'Deploying application...'
    }
}

Task with Typeâ€‹

tasks.register('copyDocs', Copy) {
    from 'src/docs'
    into 'build/docs'
}

Build Profilesâ€‹

Using Profilesâ€‹

def env = project.hasProperty('env') ? project.property('env') : 'dev'

if (env == 'prod') {
    // Production configuration

    // Development configuration
}

Run with profile:

Publishing Artifactsâ€‹

Maven Publish Pluginâ€‹

plugins {
    id 'maven-publish'
}

publishing {
    publications {
        maven(MavenPublication) {
            groupId = 'com.example'
            artifactId = 'my-library'
            version = '1.0.0'

            from components.java
        }
    }

    repositories {
        maven {
            url = "file://${buildDir}/repo"
        }
    }
}

Publish:

gradle publish

Optimization Tipsâ€‹

1. Enable Parallel Executionâ€‹

# gradle.properties
org.gradle.parallel=true

2. Enable Build Cacheâ€‹

# gradle.properties
org.gradle.caching=true

3. Increase Heap Sizeâ€‹

# gradle.properties
org.gradle.jvmargs=-Xmx2g -XX:MaxMetaspaceSize=512m

4. Use Gradle Daemonâ€‹

# gradle.properties
org.gradle.daemon=true

5. Exclude Unused Dependenciesâ€‹

configurations.all {
    exclude group: 'org.slf4j', module: 'slf4j-log4j12'
}

Troubleshootingâ€‹

Clear Gradle Cacheâ€‹

rm -rf ~/.gradle/caches/
./gradlew build --refresh-dependencies

Debug Buildâ€‹

./gradlew build --debug
./gradlew build --info
./gradlew build --stacktrace

Daemon Issuesâ€‹

./gradlew --stop          # Stop daemon
./gradlew --no-daemon     # Run without daemon

Dependency Conflictsâ€‹

./gradlew dependencies --configuration runtimeClasspath

Build Scanâ€‹

./gradlew build --scan

Best Practicesâ€‹

  1. Use Gradle Wrapper - Ensures consistent builds
  2. Enable parallel builds - Faster multi-project builds
  3. Use implementation over api - Faster compilation
  4. Version catalogs - Centralize dependency versions
  5. Minimize custom tasks - Use plugins when possible
  6. Enable build cache - Faster rebuilds
  7. Use specific versions - Avoid + in versions
  8. Keep Gradle updated - Get latest features and fixes
  9. Use buildSrc - Share build logic across projects
  10. Commit gradle wrapper - Include in version control
